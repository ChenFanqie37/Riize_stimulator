# 嫂嫂模拟器

一款 React + TypeScript 的韩娱地下恋爱生存模拟器。玩家在平行世界中成为 RIIZE 成员的秘密恋人，一边经营手机里的聊天、发帖、行程、粉圈和危机，一边守住不能被公开的关系。

> 本项目是虚构同人文游，不代表现实艺人的真实私生活、真实性格或真实经历。

## 最新玩法

- **剧情主导模式**：新增截图式文游正文推进。每轮显示数值、周/日、场景、男主、关系阶段、叙事段落，然后给出 A/B/C/D 行动。
- **双入口剧情系统**：底部“剧情”保留事件面板，“文游”进入独立沉浸阅读模式；两者共享同一个剧情回合和历史记录。
- **关键处 AI 润色**：剧情正文、选择后果、自由行动、线下安排会优先走 LLM；失败时自动使用本地模板兜底。
- **后果写回手机 App**：剧情选择会影响 KakaoTalk、Weverse、Naver、公司通知、Dispatch、相册、日程、备忘录和历史。
- **线下 App 增强**：演唱会、签售、预录、音乐节、机场、打歌下班、公司咖啡厅、品牌快闪、校园公开活动都可追。
- **“嫂子特权”但有代价**：部分线下可以让男主安排入场，通常更省钱、更甜，但会提高公司警觉、粉丝怀疑、狗仔线索或通行记录风险。
- **社交智能体评论**：粉丝、黑粉、路人、公司、狗仔、队友粉会根据帖子内容生成不同评论。
- **后端代理**：浏览器不再保存 API key，统一请求本地/服务器代理，由代理读取 `apikey.txt` 并轮换密钥。

## 核心循环

```text
创建角色
  -> 手机桌面
  -> 聊天 / 发帖 / 看粉圈 / 追线下 / 处理危机
  -> 剧情正文推进
  -> A/B/C/D 选择
  -> App 后果发酵
  -> 下一天或下一周
```

每一次发帖、聊天、约会、点赞、删帖、看热搜、追线下，都会被写入剧情钩子。后续剧情可以引用它，例如“上次那张侧门照片又被翻出来了”。

## 线下系统

线下计划有不同类别、花费、耗时、风险和甜度：

- **演唱会**：可自己买票、场外等散场，或让他安排亲友座。
- **签售**：可自己抽签/买专，或让他补名额，风险极高。
- **凌晨预录**：粉丝流程较自然，男主安排侧门会触发公司动线风险。
- **音乐节/打歌下班**：开放场地更容易偶遇，也更容易被拍背影。
- **机场**：镜头密集，任何对视都可能被剪成慢放。
- **公司咖啡厅**：最像恋爱剧情，也最容易形成固定习惯证据。
- **品牌快闪/校园活动**：相对自然，适合青春感和同款伏笔。

“让他安排”通常需要达到对应好感度和信任值。安排成功后会生成一段现场正文、一条男主消息、相册照片、证据碎片、通知和剧情钩子。

## AI 与代理

前端只请求 `/api/chat/completions`，真实 API key 留在后端。

### 密钥文件

在项目根目录创建或更新：

```text
apikey.txt
```

每行放一个 key，代理会自动读取、去重、轮换使用。多个玩家连同一个代理时，会共享这一组 key 的额度和限流池，但不会共享各自浏览器里的存档和剧情状态。

### 环境变量

复制 `.env.example`：

```bash
cp .env.example .env
```

常用配置：

```env
PORT=8787
API_BASE_URL=https://api.deepseek.com
API_MODEL=deepseek-chat
API_KEY_FILE=apikey.txt
VITE_PROXY_URL=/api
```

如果正式分享，建议设置 `PROXY_ACCESS_TOKEN`，并只把前端地址分享给朋友，不要分享 `apikey.txt`。

## 启动

安装依赖：

```bash
npm install
```

开发模式，代理和 Vite 一起启动：

```bash
npm run dev
```

只跑前端：

```bash
npm run dev:client
```

生产构建：

```bash
npm run build
```

启动后端代理并托管 `dist`：

```bash
npm run start
```

检查类型：

```bash
npm run check
```
